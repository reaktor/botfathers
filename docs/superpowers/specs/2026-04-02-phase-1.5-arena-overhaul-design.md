# Phase 1.5: Arena Visual Overhaul + Audio — Design Spec

## Overview

Transform the Phase 1 scaffold from flat colored rectangles into a cyberpunk industrial arena with animated player characters and synthwave audio. All arena textures are procedurally generated via Phaser's Graphics API. Player sprites use the existing botfather WebP assets in `assets/botfather/`.

## Player Sprites

### Source Assets
Six single-frame WebP images at 214x214px in `assets/botfather/`:
- `character_idle.webp`
- `character_front_run_run.webp`
- `character_jump.webp`
- `character_attack.webp`
- `character_hurt.webp`
- `character_death.webp`

The character faces right in all images.

### Rendering
- Load each WebP as a separate Phaser image key (`botfather-idle`, `botfather-run`, `botfather-jump`, `botfather-attack`, `botfather-hurt`, `botfather-death`)
- Scale to 48px tall (from 214px native), maintaining aspect ratio
- Physics hitbox: 32x40px, centered on sprite
- Swap texture based on player state (idle when still, run when moving, jump when airborne)
- Flip sprite horizontally (`setFlipX(true)`) when facing left; no flip when facing right (default direction)
- Per-player color tint via `setTint()`:
  - P1: Cyan `0x00ffff`
  - P2: Magenta `0xff00ff`
  - P3: Green `0x00ff66`
  - P4: Orange `0xff8800`
- Fallback: if WebPs fail to load, render as colored 48px squares (graceful degradation)

## Arena Visuals (Procedural)

All textures generated in `SpriteFactory.js` via Phaser's Graphics API.

### Color Palette
- Base dark: `#0a0a12`
- Grid/subtle lines: `#1a1a2e`
- Neon cyan: `#00ffff`
- Neon magenta: `#ff00ff`
- Neon green: `#00ff66`
- Electric blue: `#4466ff`

### Textures

**Background (`bg-panels`):** Dark industrial cyberpunk tile. Base color `#0a0a12` with subtle grid lines, panel edges, and circuit-trace details in dim purple/blue (`#1a1a2e`). Tiled across the full arena.

**Platforms (`platform-neon`):** Dark metallic fill with bright neon glowing edges (cyan/magenta gradients). Generated at a base tile size, scaled per platform width/height.

**Boundaries (`boundary-grate`):** Industrial grate pattern with neon trim. Visually heavier than regular platforms to frame the arena (floor/ceiling).

**Hole Warnings (`hole-warning`):** Neon warning strips placed at hole edges. Pulsing alpha tween animation in GameScene for visibility.

## Audio System

### AudioManager.js (new file)
- Attached to `window.AP.AudioManager`
- Uses Tone.js via CDN (`https://cdn.jsdelivr.net/npm/tone/build/Tone.min.js`)

### Sounds
- **Background track:** Synthwave loop — dark bass drone + arpeggiated synth melody, ~90 BPM. Loops continuously.
- **Ambient layer:** Low electrical hum via filtered noise, layered under the music.
- **Jump SFX:** Short synth blip triggered on player jump.

### Behavior
- Audio starts on first user interaction (click or keypress) to comply with browser autoplay policy.
- M key toggles mute/unmute for all audio.

## Fallback Strategy
- If botfather WebPs fail to load: players render as colored 48px squares
- If procedural texture generation fails: flat colors used (current behavior)
- If Tone.js CDN fails to load: game runs silently, no errors
